Learnings:
First off, AI development tools like CursorAI and Github Copilot are awesome! They have made my life way easier when working with the frontend. However, I have noticed that sometimes these tools tend to spit out "bad" code which don't run. Likewise, there have been times where the AI deletes very important code which should not be deleted! This is a major issue which caused us some pain and debugging time. Nonetheless, these tools are great, but at the end of the day, they are just tools. I found myself intervening with a lot of the code completions and having to manually fix/update code. Secondly, we should have probably used Next.js as it comes with routing out of the box and allows us to build out our apis in the same package, combining both frontend and backend. Furthermore, we could have utilized Vercel as it would have made deployments extremely simple 🤩. Lastly, we regret using Clerk since it won't work in production unless we own the domain; although for local testing, it worked extremely well and was easy to set up! Next time, we will use an alternative like Supabase Auth or Next.js Auth. Demo below ⬇️.
